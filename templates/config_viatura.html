    Código(s) do país ou estado para otimizar o reconhecimento de placas. Use br para o Brasil. Separe múltiplos com vírgula: br,ar. Deixe vazio para detecção global (menos preciso).
    Confiança mínima para aceitar uma leitura (0.0–1.0). Valores altos reduzem falsos positivos, mas podem perder placas parcialmente visíveis. Recomendado: 0.70–0.85.
    strict: só reporta quando um veículo é detectado junto — menos ruído, mais confiável.
    normal: reporta qualquer placa detectada, mesmo sem veículo confirmado — maior sensibilidade.
    Com strict, apenas placas que correspondam exatamente a um formato de região cadastrado são aceitas. Eficaz em ambientes com muito ruído visual (paredes, faixas, placas publicitárias).
    Expressão(ões) regular(es) para restringir o reconhecimento ao formato esperado. Mercosul: [A-Z]{3}[0-9][A-Z][0-9]{2}  |  Antigo: [A-Z]{3}[0-9]{4}. Separe múltiplas com vírgula.
    Analisa 1 de cada N frames do vídeo. 1 = todos os frames (máxima sensibilidade, maior CPU). 2 = 1 em cada 2 (padrão). Para alta velocidade (>80 km/h), use 1.
    Tempo máximo em segundos aguardando confirmação da placa antes de reportar o evento. Para veículos em alta velocidade, use 2–3 s. Para câmeras fixas, valores maiores dão mais tempo para o sistema confirmar a leitura.
    Intervalo mínimo em segundos entre duas leituras do mesmo veículo na mesma câmera. Evita alertas repetidos do mesmo carro. Padrão: 300 s (5 min). Para rondas móveis, 60 s pode ser mais adequado.
    plate: retorno otimizado para dados de placa — padrão para ARGOS.
    vehicle: retorno otimizado para dados do veículo — use junto com MMC ativado.
    Número máximo de placas reportadas por veículo por passagem. 1 = apenas a melhor leitura (recomendado). -1 = ilimitado.
    Restringe a detecção a uma área do frame: x_min,y_min,x_max,y_max em proporção (0.0–1.0). Exemplo: 0,0.25,1,0.85 ignora céu e capô. Deixe vazio para usar o frame completo.
    Ativa o módulo MMC. Necessário para o parâmetro speed. Aumenta uso de CPU — avalie o impacto em hardware limitado como Raspberry Pi.
    Quando ativado, veículos estacionados também geram leituras. Útil em câmeras fixas. Para viaturas em movimento, deixe desativado.
    Calcula a velocidade estimada. Requer MMC ativado. Funcionalidade beta — valores são aproximados.
    Inclui dados de trajetória no evento de dwell. Requer dwell_time ativado.
    Envia um webhook separado com o tempo total que o veículo ficou visível. Mais útil em câmeras fixas (cancelas, guaritas).
    Segundos que o sistema aguarda após o veículo sair da cena antes de disparar o evento de dwell. Evita disparos prematuros quando o veículo some por frações de segundo.
    Imagens no Webhook
    Tipo de imagem incluída no payload do webhook. vehicle é recomendado para ARGOS (recorte focado no veículo). no elimina imagens e reduz consumo de dados 4G.
    Qualidade JPEG das imagens enviadas (1–100). Menor = arquivo menor = menos dados. Para 4G, 60–75 é um bom equilíbrio entre qualidade e banda.
    Heartbeat de Câmera
    Envia estatísticas de saúde da câmera periodicamente via webhook. Diferente do heartbeat do ARGOS — este é específico do módulo LPR.
    Intervalo em segundos entre envios do heartbeat de câmera via webhook.
    Confiabilidade
    Quando ativado, webhooks que falharem são guardados em disco e reenviados assim que a conexão for restaurada. Essencial para redes 4G instáveis — mantenha ativado.
    Timeout em segundos para cada requisição ao webhook. Reduza para detectar falhas de rede mais rapidamente em 4G instável (ex: 10–15 s).
    Saída para Arquivo Local
    Padrão de nome de arquivo para salvar imagens das detecções no disco do Pi. Usa formatação strftime para datas. Deixe vazio para não salvar localmente.
    Salva um log CSV das detecções no disco. Útil como backup local quando o webhook estiver offline. Suporta strftime no nome do arquivo.
    Salva detecções em formato JSONL (uma por linha) no disco. Compatível com ferramentas de análise como pandas, jq, etc.
    Gravação de Vídeo
    Grava um clipe de vídeo para cada detecção. Requer espaço em disco no Pi. Deixe vazio para desativar a gravação de vídeo.
    Segundos gravados antes da detecção.
    Segundos gravados após a detecção.
    Taxa de frames do vídeo gravado.
    h264 gera arquivos menores mas exige suporte de hardware no Pi.
    Informações sobrepostas no vídeo gravado, separadas por vírgula. Opções disponíveis:
    all, boxes, trace, vehicle_type, dscores, confirmed, mmc, dwell_time, health_score, timestamp, position_sec, detection_zone
    Fuso horário usado nos timestamps do CSV e dos webhooks do LPR. Para Brasil: America/Sao_Paulo. Usa nomes do banco TZ (IANA).
    Intervalo em segundos de atualização das estatísticas de saúde na API interna do Stream. Não confundir com heartbeat_interval (webhook externo).
    Tempo máximo em segundos entre tentativas de reconexão ao stream da câmera. Em redes 4G instáveis, reduza para 60 s para recuperação mais rápida.
    Compensação de vibração de câmera. Deprecado a partir da versão 1.44.0 — não configure em instalações novas.
    Desativar pausa o processamento do stream sem desligar o serviço LPR. Útil para manutenção sem reiniciar o container.
    Quando ativado, o Stream reinicia se detectar alterações pela interface gráfica do Pi. Não recomendado em produção — use o botão Sincronizar do QG para controlar reinicializações.
